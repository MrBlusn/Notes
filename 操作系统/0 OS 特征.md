操作系统（Operating System, OS）是管理计算机硬件与软件资源的核心系统软件，其核心特征可概括为**并发、共享、虚拟和异步**，这四个特征相互依存，共同支撑操作系统的高效运行。

### 1. 并发（Concurrence）

**并发**是指操作系统能在==同一时间段内处理多个任务==的执行，是操作系统最基本的特征之一。

- 它==并非==指多个任务在==同一时刻同时执行==（这是 ==“并行”== 的概念，依赖多核 CPU 实现），而是通过 CPU 的==分时机制==，==让多个任务快速交替执行，从宏观上看起来像是同时进行。==
- 例如，你在电脑上同时打开浏览器浏览网页、播放音乐、编辑文档，操作系统会将 CPU 时间划分为多个时间片，轮流分配给这些程序，由于切换速度极快，用户感知不到停顿，误以为这些任务在同时运行。
- 并发的实现依赖于操作系统的进程管理、线程调度等机制，是提高 CPU 利用率的关键。

### 2. 共享（Sharing）

**共享**是指系统中的硬件和软件资源可被多个并发执行的任务共同使用，是并发存在的基础。根据资源分配方式，共享可分为两种类型：

- ==**互斥==共享**：同一时间段内，资源只能被一个任务占用，其他任务需等待该任务释放资源后才能使用。例如，打印机在打印文档时，其他程序无法同时使用，必须等待打印完成。
- ==**同时==共享**：同一时间段内，多个任务可同时访问资源（==宏观上同时，微观上可能交替==）。例如，多个程序可同时从硬盘读取数据，或访问同一个共享内存区域。
- 操作系统需通过资源管理机制（如信号量、互斥锁）协调资源共享，避免冲突和数据混乱。

### 3. 虚拟（Virtualization）

**虚拟**是指操作系统通过某种技术，将物理资源抽象为逻辑资源，为用户提供比实际物理资源更多或更便捷的使用方式。常见的虚拟技术包括：

- ==**时分复用**==：将物理资源的时间划分为多个片段，分配给不同任务使用。例如，CPU 通过分时机制，让单个 CPU “虚拟” 出多个逻辑 CPU，供多个程序同时使用。
- ==**空分复用**==：将物理资源的空间划分为多个部分，分配给不同任务。例如，内存管理中的分页、分段技术，将物理内存划分为多个逻辑地址空间，让多个程序独立使用，互不干扰。
- 典型应用如虚拟内存（将硬盘空间虚拟为内存）、虚拟机（将一台物理机虚拟为多台逻辑机），极大提升了资源的利用率和使用灵活性。

### 4. 异步（Asynchronism）

**异步**是指系统中的任务以不可预知的速度向前推进，是并发和共享带来的必然结果。
==在多道程序环境下，允许多个程序并发执行，但由于资源有限，进程的执行不是一贯到底的，而是走走停停，以不可预知的速度向前推进，这就是进程的异步性。==

- 由于多个任务并发执行，它们的执行顺序、执行时间、I/O 操作的耗时等都是不确定的，操作系统无法精确预测每个任务的执行进度。
- 例如，两个程序同时请求读取硬盘数据，由于硬盘的响应速度较慢，且可能存在优先级差异，哪个程序先完成读取、耗时多久都是不确定的。
- 但操作系统会通过进程调度、资源分配等机制，确保所有任务最终都能正常完成，不会出现无限等待的情况，即 “异步但可控”。

### 补充：四大特征的关联

并发是前提，没有并发就无需资源共享；共享是保障，为并发提供资源支持；虚拟是手段，通过抽象技术提升资源利用率；异步是结果，并发和共享导致任务执行的不确定性，而操作系统通过机制确保系统的稳定性。
