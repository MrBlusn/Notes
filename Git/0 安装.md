### 1. 安装 Git

- Windows：官网下载 [Git for Windows](https://git-scm.com/download/win)，安装后打开「Git Bash」使用命令。
- Mac：终端执行 `brew install git`（需先装 Homebrew），或官网下载。
- Linux：`sudo apt install git`（Debian/Ubuntu）、`sudo yum install git`（CentOS）。

### 2. 配置用户信息（首次使用必做）

Git 需绑定用户名和邮箱，用于标识提交者身份（建议和远程仓库账号一致）：

```bash
# 全局配置（所有仓库生效）
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱@xxx.com"

# 查看配置
git config --list
```

## 常用命令（按使用流程分类）

### 1. 初始化 / 克隆仓库

#### （1）本地新建仓库

```bash
# 进入项目文件夹
cd /你的项目路径/xxx-project
# 初始化 Git 仓库（生成 .git 隐藏文件夹，不要手动修改）
git init
```

#### （2）克隆远程仓库（下载已有仓库到本地）

```bash
# HTTPS 方式（需输入账号密码，简单）
git clone https://github.com/用户名/仓库名.git
# SSH 方式（免密，需配置 SSH 密钥）
git clone git@github.com:用户名/仓库名.git
```

### 2. 日常开发：修改 -> 暂存 -> 提交

#### （1）查看状态（最常用！随时确认修改）

```bash
git status
# 简化输出（只看关键信息）
git status -s
```

- 红色：工作区有未暂存的修改；
- 绿色：暂存区有未提交的修改；
- 无提示：工作区和暂存区一致。

#### （2）暂存修改（添加到暂存区）

```bash
# 暂存指定文件
git add 文件名.py 文件名.js
# 暂存所有修改（新增/修改/删除）
git add .
# 暂存所有修改（等价于 git add .，更通用）
git add -A
# 撤销暂存（把暂存区的文件放回工作区）
git reset HEAD 文件名
```

#### （3）提交修改（保存到本地仓库）

```bash
# 提交暂存区的修改，必须写备注（描述本次修改内容）
git commit -m "备注信息：如 修复登录按钮样式bug"

# 若忘记暂存，可直接提交所有已跟踪文件的修改（跳过暂存区）
git commit -am "备注信息"

# 修改最后一次提交（备注写错/漏加文件）
git commit --amend
```

### 3. 分支管理（核心！多人协作关键）

#### （1）查看分支

```bash
# 查看本地分支（* 表示当前分支）
git branch
# 查看本地+远程分支
git branch -a
```

#### （2）创建 / 切换分支

```bash
# 创建新分支
git branch 分支名（如 feature/login）
# 切换分支
git checkout 分支名
# 快捷：创建并切换分支（最常用）
git checkout -b 分支名
# 新版 Git 也支持：git switch -c 分支名
```

#### （3）合并分支（完成功能后合并到主分支）

```bash
# 先切换到主分支
git checkout main
# 合并目标分支到当前分支
git merge 分支名（如 feature/login）
# 解决冲突：手动修改冲突文件，然后暂存+提交
git add 冲突文件
git commit -m "解决合并冲突：登录功能"
```

#### （4）删除分支（功能合并后清理）

```bash
# 删除本地分支（确保已合并）
git branch -d 分支名
# 强制删除未合并的分支
git branch -D 分支名
# 删除远程分支
git push origin --delete 分支名
```

### 4. 远程仓库交互（同步代码）

#### （1）关联远程仓库（本地新建仓库需先关联）

```bash
# 添加远程仓库（命名为 origin，默认名称）
git remote add origin 远程仓库地址（HTTPS/SSH）
# 查看已关联的远程仓库
git remote -v
```

#### （2）拉取远程代码（同步最新版本）

```bash
# 拉取远程分支的最新代码并合并到本地当前分支
git pull origin 分支名（如 main）
# 仅拉取不合并（少用）
git fetch origin
```

#### （3）推送代码到远程

```bash
# 首次推送（需指定分支和远程仓库）
git push -u origin main
# 非首次推送（当前分支已关联）
git push origin 分支名
# 推送所有分支（少用）
git push --all origin
```

### 5. 版本回退（改错代码时用）

```bash
# 查看提交历史（按时间倒序，显示哈希值、作者、备注）
git log
# 简化日志（只看哈希值和备注）
git log --oneline
# 回退到指定版本（保留工作区修改，可重新提交）
git reset --soft 提交哈希值（如 a8b7f9）
# 回退到指定版本（删除暂存区，保留工作区）
git reset --mixed 提交哈希值（默认模式）
# 强制回退（删除工作区和暂存区所有修改，谨慎！）
git reset --hard 提交哈希值
# 恢复已回退的版本（查看所有操作记录）
git reflog
```
